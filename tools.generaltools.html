    tools.generaltools module¶

    This module contains decorators used in most of the packages of this project.

    The decorators designed in this modules are not specific to the project and are used in most of the other packages.

    list_elements(path_to_explore, type_=None, extension='', sort='natural', exception=None, verbose=False)[source]¶

    List the items in a given folder with a few filters.

    You can chose to select only one type of files: _type can be “file”, “dir”, “all”, You may want to look for files with a specific extension given by the string extension, VERBOSE makes the function print all the files found in the specified path (by default VERBOSE=False), It is possible to sort the list of files by natural order before it is returned with sort=”natural” (activated by default) or alphanumerical order with sort = “alphanumeric” You can specify a list of exception that won’t be returned with exception=[]

    Parameters:
    • path_to_explore (str) – Complete or relative path to the folder you want to explore.
    • type (str|None, optional) – Type of items you want to list, can be “dir”, “file” or None (default behavior). When set to None, the function does not care about the type and returns all items.
    • extension (str, optional) – Add a filter to the function to only return the items whose name finish with the string given to extension. By default extension is an empty string.
    • sort (str, optional) – “natural”
    • exception (list|tuple|None, optional) – List or tuple of specific items to ignore.None
    • verbose (bool, optional) – Makes the function print more information. (False by default)
    Returns:
    Return type:

    list
    Raises:

    AssertError – Raised when the types of the arguments are not respected.

    Examples: To get the list of the python files in the current directory:

    >>> list_elements(".", extension=".py")
['.\__init__.py', '.\generaldecorators.py', '.\generaltools.py']
